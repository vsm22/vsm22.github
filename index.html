Additor

a synth based on the Web Audio API

Live Version Source Code
  • JavaScript
  • Web Audio API
  • Canvas
  • RequireJS

Additor is a browser-based additive synthesizer app built in vanilla JavaScript using a custom library of components. The audio engine is built using the Web Audio API and implements a set of audio classes in a hierarchical architecture: the Overtone class represents a single oscillator; the AdditiveSynthVoice class represents a single synthesizer voice consisting of a bank of overtones; the AdditiveSynth class represents a synthesizer consisting of a bank of voices. In each level of the hierarchy the components are connected to an Envelope , which represents an audio envelope controlling the evolution of the amplitude of the component over time, specified as arbitrary arrays of time and amplitude pairs, and a ChannelStrip, which controls the panning, input gain, and output gain of the component. The UI is built using a custom library of widgets using the HTML canvas, and loosely modeled on the look of Ableton Live and Max for Live components. Module bundling is done with RequireJS.

Weathervane

weather forecast app based on the Apixu API

Live Version Source Code
  • Angular JS
  • Google Maps API
  • Apixu Weather API

Weathervane is a simple weather forecast app built with Angular JS. The app allows users to specify a location in a variety of formats and displays the current weather, and five-day and ten-day forecasts. The angular modules are organized as a component tree and use a component router to switch between the different display modes. The forecast data is received from the Apixu Weather API, and location geocoding is done with the Google Maps API. A generative background animation was created using the HTML canvas.

JS Synth UI

a mini-library of canvas-based widgets

Source Code
  • JavaScript
  • Canvas

JSLiveWidgets is a mini-library of canvas-based widgets for browser-based audio applications. The library was loosely inspired by and modeled on the look of Ableton Live and Max for Live UI components. I have been using Max extensively to prototype multimedia tools in previous projects, and have grown accustomed to having these UI components available. I thought it would be useful to have a similar set of components that are basic, customizable, and pluggable, implemented in JS for use in browser-based audio applications (and potentially in other environments as well via WebKitGtk). The project is a work-in-progress, but currently the library has the following components: keyboard, dial, slider, volume meter, envelope graph, histogram, drop-down menu, draggable number box. The library is implemented as class modules that have support for AMD and CommonJS, or can be used as globals. UI interaction is supported for mouse as well as touch events.

Ghost Dancing

music and sound design for a performance at 2016 moogfest

Audio
  • Max
  • Logic

Music and sound design for a multimedia dance performance at the 2016 moogfest with Duke's Slippage Performance Tehcnology Lab. The performance was based around two robotic installations developed by the lab, which could move and project images and live video feeds onto their surfaces. During the performance, dancers interacted with these installations in a quasi-imporvisatory structure, and invited audience members to participate. The music was created using a custom instrument based on granular synthesis built in Max and sequenced in Logic Pro.
